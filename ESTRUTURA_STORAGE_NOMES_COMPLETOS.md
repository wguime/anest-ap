## 🔄 Regras de Sanitização

Os nomes completos são sanitizados para serem compatíveis com URLs:

- **Espaços** → Hífens (`-`)
- **Acentos** → Removidos (á → a, ç → c)
- **Maiúsculas** → Minúsculas
- **Caracteres especiais** → Removidos
- **Múltiplos hífens** → Um único hífen

**Exemplos:**
- `Medicamentos de Alta Vigilância` → `medicamentos-de-alta-vigilancia`
- `Prevenção de Infecção de Sítio Cirúrgico` → `prevencao-de-infeccao-de-sitio-cirurgico`
- `KPI - Adesão aos Protocolos` → `kpi-adesao-aos-protocolos`

## 🚀 Como Usar

### Para adicionar um documento:

1. **Faça upload do arquivo** para a pasta correspondente no Storage
   - Use o nome completo sanitizado
   - Exemplo: Para `Medicamentos de Alta Vigilância`, faça upload em `medicamentos-de-alta-vigilancia/`

2. **Crie o documento no Firestore** na collection correspondente
   - Exemplo: Collection `doc_mav`

3. **Use a URL do Storage** no campo `arquivoURL` do documento

### Exemplo de URL:

```
https://storage.googleapis.com/anest-ap.firebasestorage.app/medicamentos-de-alta-vigilancia/arquivo.pdf
```

---

## 📝 Notas Importantes

1. **Arquivos `.placeholder`**: Cada pasta contém um arquivo `.placeholder` invisível que garante que a pasta exista mesmo sem arquivos.

2. **Ordem**: As pastas estão organizadas na ordem exata que aparecem no aplicativo.

3. **Nomenclatura**: Os nomes das pastas correspondem aos nomes completos exibidos no aplicativo, sanitizados para URLs.

4. **Compatibilidade**: As pastas com nomes completos são criadas além das pastas abreviadas (se existirem), permitindo organização visual melhor.
